#

피드백 트랜스포머

본 문서는 PyTorch를 사용한 접근하기: 순차 트랜스포머에서 피드백 메모리로 고수준 표현 사용하기 논문의 구현입니다.

일반 트랜스포머는 토큰을 병렬로 처리합니다. 각 트랜스포머 계층은 이전 계층의 출력에 주의를 기울입니다. 피드백 트랜스포머는 이전 단계의 모든 계층의 출력에 주의를 기울입니다. 그래서 이것은 순환성을 추가하고, 우리는 토큰을 토큰마다 처리해야 합니다. 이로 인해 훈련 속도가 크게 느려집니다(시퀀스 길이에 따라 약 5X - 10X). 그러나 예측할 때 피드백 트랜스포머는 메모리 벡터를 캐시하면 다음 토큰을 빠르게 예측할 수 있습니다.

훈련을 가속화하기 위해, 논문은 짧은 시퀀스 길이로 시작하여 점차 길이를 늘리는 것을 제안합니다. 또한 미리 훈련된 병렬 트랜스포머를 시작점으로 사용하는 것도 논의됩니다.

원래의 피드백 트랜스포머는 모든 계층의 출력을 유지하지 않습니다. 대신 모든 계층의 출력의 가중치 평균을 유지합니다. 이것은 예측 중 캐싱에 사용되는 메모리를 줄입니다. 이 파일의 처음 절반은 이것을 구현합니다.

업데이트된 피드백 트랜스포머는 키와 값 계산에 사용되는 계층 간의 가중치 Wkl​ 및 Wvl​를 공유합니다. 그런 다음 각 단계의 키와 값을 한 번만 계산하고 캐싱합니다. 이 파일의 두 번째 절반은 이것을 구현합니다. 우리는 성능 향상을 위해 사용자 정의 PyTorch 함수를 구현했습니다.

여기에 훈련 코드와 Tiny Shakespeare 데이터셋에서 피드백 트랜스포머를 훈련시키는 노트북이 있습니다.

어텐션 스코어 얻기

우리는 relative multi-head attention form Transformer-XL 논문 과 유사하게 어텐션을 위해 상대 위치 인코딩(relative positional encoding)을 사용합니다.

현재 단계 쿼리에서 j 단계의 키(현재 단계 기준)에 대한 어테션은 다음과 같습니다.

Aj​​=Q⊤Kj​=linq​(Xq+Pq​)⊤link​(Xjk​+Pj​)=(Q+UQ)⊤(Kj​+UjK​)=AQ⊤Kj​​+BQ⊤UjK​​+CUQ⊤Kj​​+DUQ⊤UjK​​​

Q,Kj​ 는 원래 임베딩 Xq,Xjk​ 과 UQ,UjK​ 는 원래 임베딩 X의 선형 변환이고, Pq​,Pj​ 는 위치 인코딩 P의 선형 변환입니다.

용어D 를 Sj​로 대체합니다.
