#

Feedback Transformer

이것은 논문 Accessing Higher-level Representations in Sequential Transformers with Feedback Memory의 PyTorch 구현입니다.

일반적인 트랜스포머는 토큰을 병렬로 처리합니다. 각 트랜스포머 레이어는 이전 레이어층의 출력에 주의를 기울입니다. 피드백 트랜스포머는 이전 단계의 모든 레이어의 출력에 주의를 기울입니다. 그래서 이것은 순환성을 추가하며, 토큰별로 처리해야 합니다. 이로 인해 훈련 속도가 크게 저하됩니다 (시퀀스 길이에 따라 약 5배 - 10배). 그러나 예측 시 피드백 트랜스포머는 메모리 벡터를 캐시하면 다음 토큰을 빠르게 예측할 수 있습니다.

훈련 속도를 높이기 위해 논문에서는 짧은 시퀀스 길이로 시작하여 점차 늘리는 것을 논의합니다. 또한 미리 훈련된 병렬 트랜스포머를 시작점으로 사용하는 것도 논의합니다.

원래의 피드백 트랜스포머는 모든 레이어의 출력을 유지하지 않습니다. 대신 모든 레이어의 출력의 가중치 합을 유지합니다. 이렇게 하면 예측 중에 캐싱에 사용되는 메모리가 줄어듭니다. 이 파일의 첫 번째 절반에서 이를 구현합니다.

업데이트된 피드백 트랜스포머는 레이어 간에 키와 값 계산에 사용되는 가중치를 공유합니다. 그런 다음 각 단계의 키와 값을 한 번만 계산하고 캐시에 저장합니다. 이 파일의 두 번째 절반쯤 에서 이를 구현합니다. 성능 향상을 위해 사용자 정의 PyTorch 함수를 구현했습니다.

여기에는 Tiny Shakespeare 데이터셋에서 피드백 트랜스포머를 훈련시키기 위한 훈련 코드와 노트북이 있습니다.
