#

Cycle GAN

이 문서는 Unpaired Image-to-Image Translation using Cycle-Consistent Adversarial Networks논문의 PyTorch 구현/지침입니다.

일부 코드는 eriklindernoren/PyTorch-GAN에서 가져왔습니다. 다른 GAN 변형도 확인하고 싶다면 참고하기 좋은 자료입니다.

Cycle GAN은 이미지를 이미지로 변환합니다. 특정 분포의 이미지를 다른 클래스 A 및 B의 이미지로 변환하도록 모델을 훈련합니다. 특정 분포의 이미지는 특정 스타일이나 성격의 이미지와 같은 것일 수 있습니다. 모델은 A와 B 사이에 쌍을 이루는 이미지가 필요하지 않으며 각 클래스의 이미지 세트만으로 충분합니다. 이는 이미지 스타일, 조명 변경, 패턴 변경 등을 변경하는 데 매우 효과적입니다. 예를 들어 여름을 겨울로, 그림 스타일을 사진으로, 말을 얼룩말로 변경합니다.

Cycle GAN은 두 개의 생성자 모델과 두 개의 판별자 모델을 학습합니다. 하나의 생성기는 이미지를 A에서 B로, 다른 하나는 B에서 A로 변환합니다. 판별자는 생성된 이미지가 실제처럼 보이는지 테스트합니다.

이 파일에는 모델 코드와 학습 코드가 포함되어 있으며 Google Colab도 있습니다.

288    def push_and_pop(self, data: torch.Tensor):
# 
290        data = data.detach()
291        res = []
292        for element in data:
293            if len(self.data) < self.max_size:
294                self.data.append(element)
295                res.append(element)
296            else:
297                if random.uniform(0, 1) > 0.5:
298                    i = random.randint(0, self.max_size - 1)
299                    res.append(self.data[i].clone())
300                    self.data[i] = element
301                else:
302                    res.append(element)
303        return torch.stack(res)

훈련

다음을 해결하는 것을 목표로 합니다: G∗,F∗=argG,Fmin​DX​,DY​max​L(G,F,DX​,DY​)

여기서 G는 X→Y로 이미지를 변환하고 F는 Y→X로 이미지를 변환합니다다. DX​는 이미지가 X공간에서 온 것인지 테스트하고, DY​는 이미지가 Y 공간에서 온 것인지 테스트합니다.

L(G,F,DX​,DY​)LGAN​(G,F,DY​,X,Y)Lcyc​(G,F)Lidentity​(G,F)​=LGAN​(G,DY​,X,Y)+LGAN​(F,DX​,Y,X)+λ1​Lcyc​(G,F)+λ2​Lidentity​(G,F)=Ey∼pdata​(y)​[logDY​(y)]+Ex∼pdata​(x)​[log(1−DY​(G(x)))]+Ex∼pdata​(x)​[logDX​(x)]+Ey∼pdata​(y)​[log(1−DX​(F(y)))]=Ex∼pdata​(x)​[∥F(G(x))−x∥1​]+Ey∼pdata​(y)​[∥G(F(y))−y∥1​]=Ex∼pdata​(x)​[∥F(x)−x∥1​]+Ey∼pdata​(y)​[∥G(y)−y∥1​]​

LGAN​ is the generative adversarial loss from the original GAN paper.

Lcyc​ is the cyclic loss, where we try to get F(G(x)) to be similar to x, and G(F(y)) to be similar to y. Basically if the two generators (transformations) are applied in series it should give back the original image. This is the main contribution of this paper. It trains the generators to generate an image of the other distribution that is similar to the original image. Without this loss G(x) could generate anything that's from the distribution of Y. Now it needs to generate something from the distribution of Y but still has properties of x, so that F(G(x) can re-generate something like x.

Lcyc​ is the identity loss. This was used to encourage the mapping to preserve color composition between the input and the output.

To solve G∗,F∗, discriminators DX​ and DY​ should ascend on the gradient,

∇θDX​,DY​​​m1​i=1∑m​​[logDY​(y(i))+log(1−DY​(G(x(i))))+logDX​(x(i))+log(1−DX​(F(y(i))))]​

That is descend on negative log-likelihood loss.

In order to stabilize the training the negative log- likelihood objective was replaced by a least-squared loss - the least-squared error of discriminator, labelling real images with 1, and generated images with 0. So we want to descend on the gradient,

∇θDX​,DY​​​m1​i=1∑m​​[(DY​(y(i))−1)2+DY​(G(x(i)))2+(DX​(x(i))−1)2+DX​(F(y(i)))2]​

We use least-squares for generators also. The generators should descend on the gradient,

∇θF,G​​m1​i=1∑m​​[(DY​(G(x(i)))−1)2+(DX​(F(y(i)))−1)2+Lcyc​(G,F)+Lidentity​(G,F)]​

We use generator_xy for G and generator_yx for F. We use discriminator_x for DX​ and discriminator_y for DY​.

585    def optimize_generators(self, data_x: torch.Tensor, data_y: torch.Tensor, true_labels: torch.Tensor):
#

훈련 모드로 변경 

591        self.generator_xy.train()
592        self.generator_yx.train()
#

Identity 손실 ∥F(G(x(i)))−x(i)∥1​ ∥G(F(y(i)))−y(i)∥1​ 

597        loss_identity = (self.identity_loss(self.generator_yx(data_x), data_x) +
598                         self.identity_loss(self.generator_xy(data_y), data_y))
#

이미지 G(x)와 F(y)를 생성합니다. 

601        gen_y = self.generator_xy(data_x)
602        gen_x = self.generator_yx(data_y)
#

GAN 손실 (DY​(G(x(i)))−1)2+(DX​(F(y(i)))−1)2 

607        loss_gan = (self.gan_loss(self.discriminator_y(gen_y), true_labels) +
608                    self.gan_loss(self.discriminator_x(gen_x), true_labels))
#

Cycle 손실 ∥F(G(x(i)))−x(i)∥1​+∥G(F(y(i)))−y(i)∥1​ 

615        loss_cycle = (self.cycle_loss(self.generator_yx(gen_y), data_x) +
616                      self.cycle_loss(self.generator_xy(gen_x), data_y))
#

Total 손실 

619        loss_generator = (loss_gan +
620                          self.cyclic_loss_coefficient * loss_cycle +
621                          self.identity_loss_coefficient * loss_identity)
#

최적화

624        self.generator_optimizer.zero_grad()
625        loss_generator.backward()
626        self.generator_optimizer.step()
#

로그 손실들 

629        tracker.add({'loss.generator': loss_generator,
630                     'loss.generator.cycle': loss_cycle,
631                     'loss.generator.gan': loss_gan,
632                     'loss.generator.identity': loss_identity})
#

생성된 이미지들을 반환합니다. 

635        return gen_x, gen_y

637    def optimize_discriminator(self, data_x: torch.Tensor, data_y: torch.Tensor,
638                               gen_x: torch.Tensor, gen_y: torch.Tensor,
639                               true_labels: torch.Tensor, false_labels: torch.Tensor):

(DY​(y(i))−1)2+DY​(G(x(i)))2+(DX​(x(i))−1)2+DX​(F(y(i)))2​

653                              self.gan_loss(self.discriminator_x(gen_x), false_labels) +

659        loss_discriminator.backward()
660        self.discriminator_optimizer.step()

로그 손실들
